# セキュリティレビューチェックリスト適用結果 — feature 005-pex-self-reject

**Date**: 2026-07-09
**対象**: PEX 破棄の良性/不審分類と `pex_rejected` 記録条件変更
**参照**: [security-review-checklist.md](../../../docs/adr/security-review-checklist.md)
(constitution 実装中ゲート 6 / Principle III)、[ADR-0013](../../../docs/adr/0013-pex-benign-rejection.md)

## 適用範囲判定

**該当**(セキュリティに関わる PR): 信頼境界コード `src/p2p/` の変更、および**ログ出力
(SecurityEvent 記録条件)の変更**を含む。

## 適用結果

- [x] **8 セキュリティイベントログ**: 本変更の中心。`pex_rejected` は**新設・改名しない**
  (カテゴリ数不変)。記録条件のみ「破棄が 1 件でも」→「**不審な破棄が 1 件以上**」へ変更。
  data-model(feature 001)§SecurityEvent の該当行と contracts/p2p-gossip.md 検査 5 を
  **先に更新**済み。良性のみは debug へ格下げし記録しない。集約・ローテーション(ログ DoS
  耐性)は不変。むしろ良性偽陽性の除去で高頻度イベントを減らし DoS 耐性は向上。
- [x] **2 入力検証(形式・内容)**: パース→形式→自己/重複→採用の多段検証と順序は**不変**。
  検証前データを後段へ渡さない挙動も不変(分類は破棄後のラベル付けのみ)。
- [x] **3 信頼境界ごとの強度**: p2p 境界の「破棄+ログ」のうち**破棄(候補登録しない防御)は
  完全に不変**(FR-004)。弱めたのはログ強度ではなく、良性(=非攻撃)事象の記録のみ。
  不審な破棄の記録は一切減らさない(混在時も記録 — FR-003)。
- [x] **4 エラー応答の情報漏洩**: `pex_rejected` の `source`/`detail` は従来どおり定型で内部情報
  なし。追加した debug ログは接続元 `source` と良性破棄件数のみ(内部状態・パス・鍵なし)。
- [x] **7 秘密情報の取り扱い**: 秘密鍵・トークンの出力なし。debug ログに載せる自ノードアドレスは
  自ノード自身のもののみ(FR-007)。ペルソナのローカル情報の送出なし。
- [x] **10 P2P: 未検証情報の不使用**: 受信 PEX アドレスの採用/破棄・候補登録の判定は不変。
  自己アドレス判定(`is_self_addr`、nonce 駆動学習)も不変。良性/不審は破棄理由のラベルのみ。
- [x] **13 テスト対応**: ネガティブシナリオを追加。単体(`src/p2p/pex.rs`)・契約
  (`tests/contract/pex.rs`: 良性のみ=無記録・重複のみ・混在=記録・回帰)・Gherkin
  (`tests/features/security.feature`: 自己反射/重複=無記録・混在=記録)。`accepted` 集合の
  回帰も各所でアサート(FR-004 / SC-004)。BDD 31 シナリオ全緑。
- [x] **14 コメント**: 分類の意図(良性=健全な網の反射・dual-stack 重複 / 不審=protocol 逸脱・
  不正入力)を `src/p2p/pex.rs`・`src/p2p/runtime.rs` に明記。
- [x] **11 P2P: 伝搬の不変条件**: 伝搬・重複抑制・再伝搬規則は本変更の対象外(PEX 候補登録の
  破棄挙動は不変)。PlusCal モデル(ADR-0005)への影響なし。
- 非該当: **1**(サイズ上限は不変)、**5**(待受・バインド・トークンの変更なし)、
  **6**(暗号処理の追加なし)、**9**(レート制限・資源上限の変更なし。分類は O(拒否件数)の
  一時ラベルのみで無制限成長構造を導入しない)、**12**(依存クレートの追加・更新なし)。

## 重点確認項目

なし(「要レビュー」該当なし)。中心の項目 8 は data-model / contract を先行更新し、記録の
減少が良性事象に限られ不審の記録を一切損なわないことを FR-003 / US2 のテストで固定済み。

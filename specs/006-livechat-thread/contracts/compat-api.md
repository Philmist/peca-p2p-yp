# Contract: 2ch 互換 API(伝統的 2ch 形式・loopback 専用)

**Feature**: `006-livechat-thread` | **参照**: research R5, spec US6 / FR-026〜FR-030

各利用者ノードが自分のためだけに提供するブリッジ。公開サーバーではない。

**参照実装(模倣対象の実勢)**: 「伝統的 2ch 形式」は bbs.jpnkn.com(BBSJPNKN —
板設定 4 項目の援用元、spec Clarifications)・EXぜろちゃんねる等の 0ch 系互換実装の
提供形式、およびそれらを読み書きする monazilla 系専ブラ・コメントビューワの取得・
書込動作に遡って解釈する。適合基準は「利用者所有の既存実況ツールが無改修で
読み書きできること」(SC-007)であり、本契約の形式詳細は実機確認で覆り得る
(001 R5 の index.txt と同型の検証方針)。実機確認の観点一覧は quickstart §5。

## 受け口・保護(FR-026)

- バインド: Settings `compat_bbs_bind`(既定 `127.0.0.1:7183`)。**loopback のみ受理**、
  非 loopback 値は起動拒否。空文字で無効化
- Host 検証: `127.0.0.1[:port]` / `localhost[:port]` 以外の Host ヘッダは定型 403
  (DNS rebinding 対策 — 001 local-api と同一方針)
- トークン認証なし(既存クライアントが送れない)。代替はレート制限 + ボディ上限(≤ 64KB)
- 違反は `compat_bbs_denied` として記録
- 文字コード: 全応答・受信フォームとも **Shift_JIS(CP932)**。SJIS へ変換不能な文字は
  **数値文字参照(`&#dddd;`)で保全**する(現代互換実装の通例。参照は `<>` を含まない
  ため区切りを壊さない — research R5)

## HTTP メタデータ・差分取得

現代の専ブラの更新チェック・差分取得が前提とする範囲を提供する:

- **Content-Type**: subject.txt / dat / SETTING.TXT / head.txt は
  `text/plain; charset=Shift_JIS`、bbs.cgi の応答ページは `text/html; charset=Shift_JIS`
- **条件付き GET**: dat・subject.txt に `Last-Modified` を付与し、`If-Modified-Since` に
  対して 304 を返す
- **差分取得**: dat への `Range: bytes=` 要求に 206 で応答する。充足不能な範囲は 416
- **内容符号化**: gzip 等は行わない(`Accept-Encoding` は無視して identity で返す)
- **dat の追記不変性 (MUST)**: 一度応答した dat のバイト列は、以後の応答で接頭辞として
  不変でなければならない(追記のみ。サイズ減少・行の書き換え = 専ブラの「あぼーん」
  再取得を発生させない)。基盤: 確定レスのみ出力・NG 非適用・名無し名はレス確定時点で
  固定(FR-023/FR-024)・エスケープ規則固定。次スレ移行・凍結後も取得済み範囲は不変

## 板の URL 対応

板ディレクトリ名 = 板 ID(スレ主ペルソナ公開鍵の hex 64 桁小文字)。
`http://127.0.0.1:7183/<board_id>/` を専ブラの板 URL として登録する
(UI にコピー可能な形で表示する)。自ノードが announce を保持している板、および
接続中・凍結中のスレを持つ板のみが解決される。

**未検証の仮定**: hex 64 桁の板ディレクトリ名を実在の専ブラが外部板として登録・巡回
できることは未検証であり、SC-007 実機確認の観点に含める(quickstart §5)。不可の場合は
短縮エイリアス(ローカル設定)の追加を検討する。

## エンドポイント

### GET /{board}/subject.txt — スレ一覧

```text
<key>.dat<>スレタイトル (レス数)\n
```

- アクティブスレ 1 行(+ 凍結スレを保持していればその行)。`key` = スレ作成 unix 秒
- 未知の板は定型 404

### GET /{board}/dat/{key}.dat — スレ本文

1 レス 1 行、**確定済みレスのみ**(FR-027。「送信中」の自分の投稿は含めない):

```text
名前<>メール<>日付 ID:xxxxxxxx<>本文<>スレタイトル\n
```

- `名前`: 空は**当該レス確定時点の**名無しのデフォルト名(FR-023/FR-024 — 追記不変性)
- `日付`: `YYYY/MM/DD(曜) HH:MM:SS`(投稿時刻。参考情報)
- `ID`: 板鍵から導出する短縮表示 8 文字(表示専用。照合には用いない — FR-018)。
  **板単位で固定であり日替わりしない**(従来の日替わり ID と異なる挙動)。専ブラの
  ID NG 機能は持続的に機能する。板内での長期追跡可能性の緩和は板鍵ローテーション
  (FR-017。ローテーション後は別 ID 表示)— spec Assumptions に記載
- **エスケープ(一意規則)**: 名前・メール・本文の `&` `<` `>` `"` をこの順で実体参照
  (`&amp;` `&lt;` `&gt;` `&quot;`)へ変換し、本文の改行は `<br>` に変換する。
  アンカー `>>n` は変換の結果 `&gt;&gt;n` として出力される(専ブラ側が解釈する従来形式)。
  SJIS 変換不能文字は数値文字参照 `&#dddd;`(§受け口)。dat から本文への逆変換は
  行わない(エスケープは dat 出力時のみの表現であり、正本はイベントの生テキスト)
- `スレタイトル`: 1 レス目の行のみ。2 行目以降は空
- NG は互換 API には適用しない(欠番表示は UI の責務。dat は確定全レスを返す)
- 凍結・クローズ後も取得済み分は返す。**保持していない dat(クローズで削除済み・
  過去世代の未保持 key・未知の key)は定型 404** — 専ブラは非 200 を「dat 落ち」相当と
  して扱う想定(SC-007 の確認観点)

### GET /{board}/SETTING.TXT — 板設定提示(FR-027)

```text
BBS_TITLE=<板タイトル>
BBS_NONAME_NAME=<名無しのデフォルト名>
BBS_LINE_NUMBER=32
BBS_MESSAGE_COUNT=2048
BBS_NAME_COUNT=64
BBS_MAIL_COUNT=64
```

- 単位はいずれも**文字数**(従来慣行)であり、イベントスキーマの上限
  (本文 ≤ 2048 文字・名前/メール ≤ 64 文字 — thread-events.md)と同一値・同一単位。
  SETTING.TXT を尊重した専ブラの書き込みがホスト検証で拒否される単位齟齬は生じない
- 板設定 4 項目(spec Clarifications — BBSJPNKN 援用)との対応: 板タイトル → `BBS_TITLE`、
  名無しのデフォルト名 → `BBS_NONAME_NAME`、ローカルルール → head.txt(下記)。
  スレ最大レス数は従来 SETTING.TXT に標準キーがないため拡張キー
  `BBS_MAX_RES=<res_limit>` で出力する(未知キーを無視するクライアントに影響しない)。
  jpnkn / EX0ch 系の提示キー集合との突合は SC-007 の確認観点

### GET /{board}/head.txt — ローカルルール

板設定のローカルルール(Markdown)を**平文のまま** Shift_JIS で返す(描画しない —
research R7)。既存クライアントが head.txt を HTML として描画する場合、Markdown 記号が
生のまま見えるが、これは受容する(専ブラはプレーンテキスト表示が通例 — SC-007 の
確認観点)。

### POST /test/bbs.cgi — 書き込み(FR-028)

- フォーム(Shift_JIS URL エンコード): `bbs`(board_id)、`key`、`time`、`FROM`(名前)、
  `mail`、`MESSAGE`(本文)、`submit`
- **書き込み前段なし**: クッキー確認画面(初回書き込み時の同意フロー)は返さない —
  常に直接受理または直接エラーとする。専ブラの確認画面自動応答フローは発動しないが、
  成功/エラーの `<title>` 判定のみで完結するため影響しない(SC-007 の確認観点)。
  Cookie は発行・要求せず、Referer 検査は行わない(loopback 限定のため不要 — FR-026)
- **数値文字参照の展開**: `MESSAGE`・`FROM`・`mail` 中の数値文字参照
  (`&#dddd;` / `&#xhhhh;`)は受理時に展開して本文(UTF-8)へ格納する(専ブラが
  SJIS 外の文字を数値文字参照で送る通例への対応)
- **スレ立て不可**: `subject` パラメータ付き(新規スレ作成)の POST は従来形式の
  定型エラーで拒否する(スレの開設は配信者の明示操作のみ — FR-001 / spec Edge Case)
- 処理: 名前欄の `#` 以降除去 → 板鍵で自動署名(なければ生成し初回 PoW を計算)→
  **通常経路と同一の検証・送信**(thread-events.md のホスト側検証をそのまま経由。
  互換経路だけの抜け道を作らない — FR-028)
- 成功応答: `<title>書きこみました。</title>` を含む従来形式の HTML ページ
- エラー応答: `<title>ERROR!</title>` + `ERROR:<定型メッセージ>` の従来形式。
  内部情報(パス・スタックトレース・BAN の事実)を含めてはならない (MUST NOT)
  (FR-030。凍結・クローズ・満員・レート違反は従来クライアントが解釈できる定型文言で返す)
- 採番確定は非同期のため、成功応答は「ホストへの送信受理」を意味する。反映は
  dat 再取得で確認される(US6 シナリオ 3)

# ADR-0011: 配信中ペルソナロック(掲載前の選択とリンク推定の構造的防止)+ Principle V 判定

**Status**: Accepted
**Date**: 2026-07-07
**Principles**: Principle I (Safety First), Principle II (Security by Design), Principle V (Formal Verification for Critical Paths), Principle VI (Principle Traceability)
**Feature**: [specs/003-persona-selection](../../specs/003-persona-selection/spec.md)

## 背景

ユーザーがチャンネルを掲載(kind 30311 発行)する**前に**、名乗るグローバル「選択中ペルソナ(selected)」を能動的に切り替える機能を追加する(spec US1)。現状 selected を切り替える UI が存在せず、選択機会が掲載前の設定に限られる。

この選択機能を安全にする核心は、**配信中に selected を入れ替えさせないこと**である。掲載エンジンは周期再発行のたびに署名ペルソナを selected から再解決するため、配信中に selected が変わると同一 ChannelID 上で「旧ペルソナ `ended` → 新ペルソナ `live`」が観測され、[ADR-0004 §7](./0004-threat-model.md) が挙げる**ペルソナ間リンク推定の強いシグナル**を生む。複数ペルソナを使い分ける唯一の目的(同一人物と結び付けられないこと)を根本から破壊するため、Principle I の要請として構造的に防ぐ。

## 決定

### 1. 配信中ロック(selected の凍結)

「配信中」を **1 つ以上のチャンネルが実際にネットワークへ発行されている状態**と定義する(揮発の共有状態 `BroadcastState` の配信中集合が非空)。保留中(未発行)チャンネルは含めない(spec FR-008)。配信中は、selected ペルソナに対する以下を拒否する(HTTP 409 `broadcasting_locked`):

- 別ペルソナへの selected 切替
- selected ペルソナの破棄(delete)
- selected ペルソナのアーカイブ(active→archived)

一方、以下は配信中でも許可する:

- selected ペルソナの label(表示名)変更 — 署名アイデンティティに影響せずリンクを生まない(FR-006)
- 配信に無関係な他ペルソナの作成・破棄・アーカイブ — ロック対象は selected のみ(FR-007)

これにより selected は配信中に凍結され、実署名ペルソナと解決結果が常に一致する。掲載一覧の真実源切替や掲載エンジンのペルソナピン留め改修は不要になる(ピン方式を棄却した理由は spec 設計インタビュー / research R2 参照)。

**セキュリティログ非該当の判定(Principle I/II・§Security Requirements)**: 本機能の拒否(`broadcasting_locked` / `persona_not_selectable` の 409)は、既存 001 のローカル API 保護(Host 検証・トークン)を通過した**ローカル正当利用者**による業務ルール上の拒否であり、constitution §Security Requirements が「不正なリクエスト」として記録を求めるネットワーク越しの攻撃イベントには当たらない。新規のネットワーク受信経路も追加しないため、これら拒否はセキュリティログの対象外とする(必要なら `debug!` レベルの非機微な記録を任意で許容するが必須としない)。**再評価トリガ**: リモート/非ローカルからの selected 変更経路を将来追加する場合は本判定を見直す。

### 2. TOCTOU レースの構造的閉塞(予約先行)

`select()` と発行開始の間に発行が割り込むと、「配信中判定=空」を見た切替が通った直後に発行が始まり、次回再発行で入替が生じうる。これを構造的に防ぐため、`BroadcastState` の単一ミューテックスで 2 つのクリティカルセクションを相互排他にする:

- **発行開始**(あるチャンネルの初回発行): ロック下で **selected を読み取り + 当該チャンネルを配信中集合へ予約**した後にロックを解放し、その後で署名・送信する。**予約は署名より先**に行う(署名=暗号処理はロック外・失敗時は予約を巻き戻す)。
- **selected の変更/破棄/アーカイブ**: 同じロックを取り、配信中集合が非空(対象が selected)なら拒否する。

どちらのクリティカルセクションが先に成立しても不変条件「配信中の区間、当該チャンネルの署名ペルソナは変化しない」が保たれる(research R2 の場合分け)。

### 3. 選択可能条件と selected セマンティクス

- `select()` は **active かつ usable(鍵復号可能)**なペルソナのみ許可する。archived は 409 `persona_not_selectable`、復号不可は 422 `persona_unusable`(FR-002、UI のみの防御にしない)。
- `selected()` は、対象が破棄済み・**archived・unusable** のいずれでも未選択(None)相当を返す。この場合 UI は警告表示・再選択導線を出し、掲載は保留に落ちる(FR-011)。selected が配信中に外部要因(鍵消失)で unusable 化する事象は利用者操作ではないためロック対象外で、保留フォールバックに委ねる。

### 4. 共有状態の所有(循環依存の回避)

`BroadcastState` を中立な新規モジュール `src/broadcast.rs` に置き、`IdentityManager`・`PublishEngine`・Web の `AppState` が各々 `Arc<BroadcastState>` を保持する(相互に相手を所有しない)。`PublishEngine` が `IdentityManager` を所有する既存関係を保ったまま、両者が同一の配信中状態を原子的に共有できる(research R3)。

## Principle V(形式的検証)判定: **非該当**

constitution Principle V のクリティカル 3 基準(すべて充足で該当)を判定する。

| 基準 | 判定 | 根拠 |
|------|------|------|
| ① 新規設計であり、既存の実績ある仕様・ライブラリの単純な利用ではない | **不成立** | 中核は単一の共有ミューテックスによる 2 クリティカルセクションの相互排他。標準的な並行プリミティブ(`Mutex`)の単純利用であり、[ADR-0005](./0005-gossip-formal-verification.md) の gossip 伝搬のような創発的性質を持つ自前プロトコルではない |
| ② 競合状態がテストで再現困難な非自明さを持つ | **限定的** | 相互排他が成立すれば不変条件は構成的に真になる。予約先行(決定 §2)により振る舞いは 2 通りの場合分けに収まり、少数ノード・少数チャンネルの統合テストで意図的に交錯を再現できる |
| ③ 失敗がユーザー安全(Principle I)またはデータ整合性に直接影響する | **成立** | 配信中のペルソナ入替はリンク推定シグナル(§背景)を生み、プライバシー = Principle I に直接影響する(spec SC-005) |

基準①が不成立のため **非該当**。TLC による網羅探索の限界価値は低い(相互排他の正しさは自明)。

**代替担保(Principle V は SHOULD、理由記録で非該当を許容)**: 予約→署名の順序と配信中の切替拒否を検査する**並行性統合テスト**を必須とする(select と発行開始を交錯させ、いずれの順でも不変条件が保たれることを確認)。加えて cucumber セキュリティシナリオで直接 API バイパス試行(配信中の切替/破棄/アーカイブ)が 409 で拒否されることを検証する(Principle IV)。

**再評価トリガ**: ロック方式を多ロック化・ペルソナ再解決方式・per-channel 割当の導入等へ変更する場合は本判定を再評価し、必要なら PlusCal モデル(select vs publish-start の相互排他)を `docs/formal/` に追加する。

## 帰結・残余リスク

- 配信中は selected を凍結するため、配信を止めない限りペルソナを替えられない。これは UX 上の制約だが、リンク推定防止(Principle I)が優先する。停止後は追加操作なしにロックが解ける(FR-009)。
- selected の切替は非破壊操作であり確認ダイアログを設けない(FR-012)。誤爆防止は常時の明示表示(personas.html バナー + channels.html の読み取り専用表示)で担保する。既存の破棄・nsec エクスポートの確認は維持する。
- トラッカー IP による直接リンク([ADR-0004 §7]・§8)は本ロックの対象外(プロトコル/ローカル操作の外側)。本 ADR はプロトコル層のイベント入替シグナルのみを扱い、匿名性保証の限界は ADR-0004 の記載を正とする。
